# Agentes y Memoria EPO 67

Esta carpeta reduce lecturas repetidas y gasto de contexto.

## Orden recomendado

1. Leer `.agents/memory/PROJECT_STATE.md`.
2. Leer solo el brief de `.agents/agents/` relacionado con la tarea.
3. Revisar los archivos de código concretos.
4. Actualizar `PROJECT_STATE.md` al terminar cambios relevantes.

## Regla práctica

No abrir documentación larga salvo que el brief lo pida o haya duda real. La fuente de verdad sigue siendo el código y `AGENTS.md`.

## Mapa rápido

- `memory/PROJECT_STATE.md`: estado vivo del proyecto y cambios recientes.
- `memory/DECISIONS.md`: decisiones tomadas para no rediscutirlas.
- `agents/system-builder.md`: desarrollo del frontend/app.
- `agents/report-generator.md`: impresión y reportes.
- `agents/data-validator.md`: validación/auditoría de datos.
- `agents/security-auditor.md`: seguridad y reglas.
- `agents/data-manager.md`: scripts y migraciones.
